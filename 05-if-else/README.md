# 05 - If/Else 条件判断

## 概述

条件判断是最基础的控制流结构。三种语言在 if/else 的设计上有明显差异，尤其是 Zig 和 Rust 中 `if` 是表达式（可以返回值），而 Go 中 `if` 是语句。

## 核心差异

| 特性 | Zig | Go | Rust |
|------|-----|-----|------|
| if 是表达式 | 是 | 否 | 是 |
| 三元运算符 | 用 if 表达式代替 | 无，需用 if 语句 | 用 if 表达式代替 |
| 条件需要括号 | 不需要 | 不需要 | 不需要 |
| 大括号是否必须 | 单表达式可省略 | 必须 | 必须 |
| Optional 解包 | `if (opt) \|val\|` | 无（用 nil 检查） | `if let Some(val)` |
| 条件中声明变量 | 不支持 | 支持 | 不支持 |

## Zig 特色

1. **if 是表达式**：可以直接把 if 的结果赋值给变量，类似其他语言的三元运算符。
2. **Optional 捕获**：Zig 的 `if` 可以从 optional 类型中捕获值（`if (opt) |val|`），这是 Zig 处理可空值的惯用方式。
3. **不需要括号**：条件表达式不需要用小括号包裹。

## Go 特色

1. **条件前可声明变量**：`if err := doSomething(); err != nil { ... }` 是 Go 的经典模式。
2. **没有三元运算符**：Go 有意不提供三元运算符，认为 if/else 更清晰。
3. **大括号必须**：即使只有一行也要大括号。

## Rust 特色

1. **if 是表达式**：和 Zig 类似，if 可以返回值。
2. **if let 解构**：`if let Some(val) = opt` 用于模式匹配解构。
3. **所有分支类型必须一致**：作为表达式时，各分支必须返回相同类型。

## 运行方式

```bash
# Zig
zig run main.zig

# Go
go run main.go

# Rust
rustc main.rs -o main_rs && ./main_rs
```
