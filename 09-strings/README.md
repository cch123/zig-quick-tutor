# 09 - 字符串 (Strings)

字符串处理是每种语言的核心功能之一。Zig、Go 和 Rust 在字符串的设计理念上有着显著差异。

## 核心概念

### Zig
Zig 没有专门的字符串类型。字符串本质上就是 `[]const u8`（只读字节切片）。字符串字面量在编译期已知，类型为 `*const [N:0]u8`（以 null 结尾的指针，方便与 C 互操作）。这种设计简单直接，避免了隐藏的内存分配。

### Go
Go 的 `string` 是不可变的字节切片。底层由一个指针和长度组成。`rune`（即 `int32`）用于表示 Unicode 码点。字符串拼接会产生新的分配。

### Rust
Rust 有两种主要字符串类型：`String`（堆分配、可增长）和 `&str`（字符串切片引用）。两者都保证 UTF-8 编码。

## 对比表

| 特性 | Zig | Go | Rust |
|------|-----|-----|------|
| 字符串类型 | `[]const u8` | `string` | `String` / `&str` |
| 编码 | 原始字节 | 原始字节 | 保证 UTF-8 |
| 可变性 | 取决于切片类型 | 不可变 | `String` 可变 |
| 编译期拼接 | `++` 运算符 | 不支持 | 不支持 |
| 运行时拼接 | `std.mem.concat` | `+` / `strings.Builder` | `format!` / `push_str` |
| 格式化 | `std.fmt.bufPrint` | `fmt.Sprintf` | `format!` |
| 多行字符串 | `\\` 续行 | 反引号 `` ` `` | `r#"..."#` 或普通多行 |
| Null 结尾 | 字面量自动带 `\0` | 否 | 否 |

## 要点

1. **Zig 的 `++` 只能用于编译期已知的值**：运行时拼接需要分配器。
2. **Go 的字符串拼接很方便但有性能陷阱**：大量拼接应使用 `strings.Builder`。
3. **Rust 严格区分拥有和借用**：`String` 拥有数据，`&str` 只是引用。
4. **Unicode 处理**：三种语言都支持 UTF-8，但 Zig 需要手动使用 `std.unicode.utf8` 模块。
