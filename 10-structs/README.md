# 10 - 结构体 (Structs)

结构体是组织数据的基本方式。三种语言在结构体设计上各有特色。

## 核心概念

### Zig
Zig 的结构体功能丰富：支持默认值、方法（通过命名空间声明）、`packed` 和 `extern` 布局控制。方法的第一个参数显式传入 `self`。Zig 还支持匿名结构体和元组。

### Go
Go 的结构体简单直接，使用零值初始化。方法通过接收者函数（receiver）定义。没有构造函数语法，惯用法是 `NewXxx()` 工厂函数。

### Rust
Rust 的结构体通过 `impl` 块定义方法。没有默认值（但可以实现 `Default` trait）。支持元组结构体和单元结构体。所有权系统影响结构体的使用方式。

## 对比表

| 特性 | Zig | Go | Rust |
|------|-----|-----|------|
| 方法定义 | 结构体命名空间内 | 接收者函数 | `impl` 块 |
| self 参数 | 显式 `self`/`*self` | 显式接收者 | `self`/`&self`/`&mut self` |
| 默认值 | 字段级默认值 | 零值 | `Default` trait |
| 内存布局控制 | `packed`/`extern` | 无保证 | `#[repr(C)]`/`#[repr(packed)]` |
| 匿名结构体 | 支持 | 支持 | 不支持 |
| 元组结构体 | 支持（元组） | 不支持 | 支持 |
| 构造函数 | `init()` 惯用法 | `NewXxx()` 惯用法 | `new()` 惯用法 |
| 嵌入/组合 | 无语法糖 | 匿名嵌入 | 无（用 trait） |
| 可见性 | `pub` 关键字 | 首字母大写 | `pub` 关键字 |

## 要点

1. **Zig 的 `self` 是显式的**：类似 Python，方法的第一个参数就是实例本身。
2. **Go 的零值很方便**：所有字段自动初始化为零值，不需要构造函数。
3. **Rust 的所有权影响方法签名**：`self` 消耗实例，`&self` 借用，`&mut self` 可变借用。
4. **Zig 的 packed struct**：可以精确控制内存布局，适合底层编程和协议解析。
