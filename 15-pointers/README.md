# 第十五章：指针（Pointers）

指针是系统编程的核心概念。三种语言对指针的抽象层次和安全保证各不相同。

## Zig 的方式

Zig 提供多种指针类型，保留了底层控制能力：

- `*T`：单元素指针
- `[*]T`：多元素指针（类似 C 指针，支持指针算术）
- `[*:0]u8`：哨兵终止指针（如 C 字符串）
- `*allowzero T`：允许零值的指针
- `@ptrCast`：指针类型转换
- `@alignCast`：对齐转换

Zig 允许指针算术，但通过切片（slice）提供了更安全的替代方案。

## Go 的方式

Go 的指针经过刻意简化：

- `*T`：指针类型
- `&v`：取地址
- 没有指针算术（除非用 `unsafe` 包）
- 自动解引用（方法调用时）
- 垃圾回收管理内存

## Rust 的方式

Rust 通过所有权和借用系统提供内存安全：

- `&T`：不可变引用（共享借用）
- `&mut T`：可变引用（独占借用）
- `Box<T>`：堆分配的拥有型指针
- `*const T` / `*mut T`：原始指针（仅在 `unsafe` 中使用）

## 对比表

| 特性 | Zig | Go | Rust |
|------|-----|-----|------|
| 单指针 | `*T` | `*T` | `&T` / `&mut T` |
| 多元素指针 | `[*]T` | 无 | 无（用切片） |
| 指针算术 | 支持 | 不支持 | 仅 `unsafe` |
| 空指针 | 不允许（需 `?*T`） | `nil` | 不允许（用 `Option`） |
| 自动解引用 | 否 | 部分（方法调用） | 是（Deref trait） |
| 类型转换 | `@ptrCast` | `unsafe.Pointer` | `as *const T` |
| 内存管理 | 手动 | GC | 所有权系统 |
| 哨兵指针 | `[*:0]u8` | 无 | `CStr` |
| 堆分配 | 手动 allocator | `new()` (GC) | `Box::new()` |

## 运行方式

```bash
zig run main.zig
go run main.go
rustc main.rs -o main_rs && ./main_rs
```
